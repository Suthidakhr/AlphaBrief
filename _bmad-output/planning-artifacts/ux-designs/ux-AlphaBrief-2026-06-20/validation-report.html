AlphaBrief — UX Design Validation Report

_bmad-output/planning-artifacts/ux-designs/ux-AlphaBrief-2026-06-20/ · 3 reviewers · 2026-06-20

The spine pair is implementation-ready with targeted fixes required before development begins. DESIGN.md is structurally complete and follows canonical section order; EXPERIENCE.md covers all three user journeys with named protagonists, numbered steps, and climax beats. The critical issues split across two domains: four color contrast combinations fail WCAG 2.1 AA (the most pervasive being camel #B2967D used as text on white at 2.39:1), and two compliance gaps — the Theme Detail page has no AI disclaimer, and the Daily Brief card's disclaimer is specified as a navigation link rather than inline text, both violating FR-A04.

Five sidebar and page-level components (MarketOverviewWidget, SectorHeatmap, TrendSummary, News Detail page, Theme Detail page) have no behavioral spec and appear only in the IA tree. All three working mockups are orphaned — referenced nowhere in either spine. These are high-severity implementation blockers but require additions, not corrections. The color token architecture is sound; the contrast failures are in how tokens are assigned to text roles, not in the token values themselves. Fixing the text-role assignments (camel → cocoa for body text) and darkening the positive sentiment token resolves the contrast chain without changing the palette identity.

1. Flow coverage

Adequate

EXPERIENCE.md §Key Flows / Flow 2

Accessibility review (WCAG 2.1 AA)

review-accessibility.md

Accessibility intent is strong (non-color indicators, keyboard nav, landmarks, touch targets, motion preferences all addressed). Four contrast combinations fail WCAG 1.4.3; the camel-on-white failure (2.39:1) is pervasive because camel is assigned to text roles throughout the system. The fix is architectural: reassign text roles to cocoa (#7D5A44, ~5.1:1 on white) and reserve camel for accent/decorative use only.

Critical

Camel (#B2967D) on white (#FFFFFF) — 2.39:1, fails 4.5:1

DESIGN.md §Colors/Accent Scale, §Typography, §Components (section labels, metadata, nav sub-labels)
Used pervasively: section labels (12px 600), source names, timestamps, AI insight box border area, category filter tabs.
Fix: Designate camel as an accent color only (decorative borders, icons, large heading accents). Shift all normal-size text roles to cocoa (#7D5A44, ~5.1:1 on white). Update type scale table in DESIGN.md to specify cocoa for label/metadata text roles.
Critical

Camel (#B2967D) on linen-deep (#EDE8E0) — 2.01:1, fails 4.5:1

DESIGN.md §Colors, §Typography metadata row
Fix: Use neutral-text (#78716c, 4.30:1 on linen-deep) or cocoa (#7D5A44, ~5.0:1 on linen-deep) for page-level metadata text on the deep linen background.
Critical

Analysis pending state — camel at 60% opacity on linen (~1.6:1)

EXPERIENCE.md §Analysis Pending
The pending-state text is specified as "camel at 60% opacity" — completely inaccessible. Opacity-based text must never be used for meaningful content states.
Fix: Use cocoa (#7D5A44) at full opacity for pending-state text. The muted feel can be achieved through font-weight (400 vs 700) rather than opacity.
High

Positive (#16a34a) on positive-bg (#dcfce7) — 4.13:1, fails 4.5:1

DESIGN.md §Colors/Sentiment Colors, §Components/SentimentBadge, tailwind.config.ts
Badge text at 9–11px bold doesn't qualify as large text; needs 4.5:1. Margin is small (0.37:1).
Fix: Darken positive token to #15803d (Tailwind green-700, ~5.0:1 on #dcfce7). Update tailwind.config.ts and DESIGN.md frontmatter.
High

Neutral-text (#78716c) on white (#FFFFFF) — 4.30:1, fails 4.5:1

DESIGN.md §Typography metadata row
Used for source names, timestamps, body text. Fails by 0.20:1. Borderline but technically non-compliant.
Fix: Darken neutral-text to #6b6560 (~4.6:1 on white). Update token in DESIGN.md frontmatter and tailwind.config.ts.
High

Focus ring (camel at 35% opacity) ~1.3:1 — fails WCAG 1.4.11 UI component contrast (3:1)

DESIGN.md §Elevation, EXPERIENCE.md §Accessibility Floor
Fix: Replace with espresso-based focus ring: "box-shadow: 0 0 0 2px #4A342A" with a 2px white offset (box-shadow: 0 0 0 2px white, 0 0 0 4px #4A342A). Espresso on white = 12.5:1 — exceeds 3:1 on any AlphaBrief surface.
High

Stock badges — no aria-label pattern specified

DESIGN.md §Components/NewsCard, EXPERIENCE.md §Component Patterns/NewsCard
Screen readers will announce "▲ PTT" as "triangle up PTT" — not "PTT: rising".
Fix: Specify aria-label pattern: aria-label="PTT: rising" / "AAV: falling" / "DELTA: unchanged".
High

Search input — no label specification (placeholder only)

EXPERIENCE.md §Search
Fix: Add aria-label="Search news, stocks, and sectors" to the search input spec. Placeholder text disappears on focus and does not substitute for a label.
Medium

SectorHeatmap and MarketOverview — no non-color indicators defined

DESIGN.md §Colors/Sentiment Colors
Fix: Heatmap cells must show percentage text (e.g., "+1.87%") alongside color fill. MarketOverview direction indicators must use ▲/▼/– arrows in addition to color. Define in new component specs.
Medium

AI insight box — no accessible name for screen readers

DESIGN.md §Components/AIInsightBox
Fix: Add a visually-hidden heading or aria-label="AI market analysis" on the insight box container element.
Medium

Category filter tab row — no ARIA role specification

EXPERIENCE.md §Category Filter Bar
Fix: Specify role="tablist" on container, role="tab" + aria-selected="true/false" on each tab, left/right arrow key navigation within the group.
Medium

Thai subtitle text — effective contrast ~3.5:1 at 70% opacity on espresso

DESIGN.md §Typography (Thai subtitle row), §Components/Navbar, §BottomTabBar
Fix: Mark Thai subtitles aria-hidden="true" (they duplicate the English label already read aloud) and increase opacity to ≥90% visually.

Analysis pending dot animation not listed under prefers-reduced-motion

EXPERIENCE.md §Analysis Pending

Financial compliance review

review-compliance.md

Sentiment color enforcement (data-layer), error/empty state rules (explicit timestamped messages), staleness thresholds, and the financial language prohibition are all well-covered. Two critical gaps: Theme Detail page has no AI disclaimer (FR-A04 requires it on every analysis surface); and the Daily Brief card spec describes a "disclaimer link" rather than inline text, which does not satisfy FR-A04's structural rendering requirement.

Critical

Theme Detail page — no AI disclaimer specified

EXPERIENCE.md §IA (/trends/[id]), EXPERIENCE.md §Component Patterns (ThemeCard)
FR-A04: every surface rendering AI-generated analysis must include the disclaimer as non-removable. Theme Detail renders both the theme description and constituent article analyses.
Fix: Add "AI disclaimer (non-removable)" to the Theme Detail IA entry and to the ThemeCard component spec for the detail page view.
Critical

Daily Brief card disclaimer specified as "link" — must be inline text

DESIGN.md §Components/DailyBriefCard
DESIGN.md DailyBriefCard footer reads "disclaimer link" — implying a hyperlink to About. FR-A04: if analysis renders, the disclaimer renders. A navigation link does not satisfy this.
Fix: Change to "inline disclaimer text (non-removable)" and quote FR-D05 canonical string verbatim.
High

News Detail source attribution ambiguous — publisher name not explicitly required

EXPERIENCE.md §IA (News Detail surface)
IA entry says "Headline + source link" — a bare hyperlink on the headline would satisfy "source link" but violate NFR-D01 (publisher name non-nullable).
Fix: Update to: "Headline + source name (non-nullable, visible text) + link to original article (non-nullable)."
High

Staleness indicator not specified for Theme Detail constituent article analyses

EXPERIENCE.md §Component Patterns / ThemeCard (detail view)
FR-A06 applies to all AI analysis displays. Constituent articles on a Theme Detail page may have stale analyses; no indicator is specified.
Fix: Add staleness rule to constituent article rendering on Theme Detail: same 24h amber indicator as on the News Detail page.
Medium

Canonical disclaimer strings not quoted verbatim in either spine

EXPERIENCE.md §Voice and Tone / Microcopy, DESIGN.md §Components/Disclaimer
FR-A04 and FR-D05 have slightly different canonical strings. EXPERIENCE.md microcopy matches FR-A04 but not FR-D05. Developers should copy-paste, not paraphrase.
Fix: Add a "Required disclaimer strings" note to EXPERIENCE.md quoting both verbatim, labeled by FR reference.
Medium

Financial language prohibition does not bind AI system prompt

EXPERIENCE.md §Design Philosophy
The prohibition covers UI microcopy but neither spine references NFR-AI01 (AI system prompt must enforce these prohibitions and be version-controlled as application code).
Fix: Add a line in Design Philosophy: "These prohibitions apply equally to AI-generated analysis text. The AI system prompt (see NFR-AI01) is the enforcement mechanism for the AI output layer."
Low

Sentiment inversion prohibition is implicit, not explicit

DESIGN.md §Do's and Don'ts
Fix: Add: "Don't: Invert the green/red sentiment mapping under any circumstance, including in success states, loading indicators, or non-market UI contexts."

Mechanical notes

  • Nav label / route split: Nav reads "Stocks / หุ้น"; surface and component docs read "Trends". Unresolved before stories are written = divergent implementation. Pick one.
  • Bottom tab bar 4th tab ambiguous: If "Stocks" maps to Trends (themes), the 4th tab may be About — but About has its own nav in the desktop navbar. Confirm the mobile tab bar structure.
  • WCAG 2.1 vs. 2.2: Spines commit to WCAG 2.1 AA (matching the PRD). WCAG 2.2 is current (Oct 2023). Consider upgrading for a new product if launch is planned post-2024.
  • UJ naming prefix divergence: PRD "UJ-1" vs. EXPERIENCE.md "Flow 1". Low risk (protagonist names align) but creates lookup friction for story-writers tracing coverage.
  • Typography token gap: No {typography.*} path tokens in DESIGN.md frontmatter. Prose table is sufficient since EXPERIENCE.md makes no {typography.*} path references. No action required unless token refs are added.
